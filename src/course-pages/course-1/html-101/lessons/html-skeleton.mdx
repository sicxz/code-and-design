## What Every Page Has in Common

- Open any website's source code — news, portfolio, banking app — same structure at the top
- Not the same content, not the same design — the same **skeleton**
- Browsers expect this structure to be present and in the right order

<div class="lesson-note">
  <h4>Also called: Boilerplate</h4>
  <ul>
    <li><strong>Boilerplate</strong> = standard code you copy as a starting point for every new project, with little or no modification</li>
    <li>The term comes from printing — boilerplate plates were reused blocks of standard text</li>
    <li>In web dev, "HTML boilerplate" means this exact skeleton — you'll type or paste it at the start of every new page</li>
    <li>Many editors (VS Code, <a href="https://codepen.io/" target="_blank" rel="noopener">CodePen</a>) generate it automatically when you type <code>!</code> and hit Tab</li>
  </ul>
</div>

## The Required Structure

Every line, annotated:

<div class="annotated-skeleton" role="table" aria-label="HTML skeleton structure breakdown">
  <div class="annotated-code" role="cell">{'<!DOCTYPE html>'}</div>
  <div class="annotated-label" role="cell">Tells the browser: this is HTML5</div>

  <div class="annotated-code" role="cell">{'<html lang="en">'}</div>

  <div class="annotated-code i2" role="cell">{'<meta charset="UTF-8">'}</div>

## A More Complete Example

Here's the skeleton with real content added — notice the hierarchy stays intact:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Ramen Recipe — Cooking Lab</title>
  </head>
  <body>
    <h1>Classic Shoyu Ramen</h1>
    <p>A rich, clear broth with soy sauce and toppings.</p>

    <h2>Ingredients</h2>
    <ul>
      <li>4 cups chicken stock</li>
      <li>2 tbsp soy sauce</li>
      <li>2 portions ramen noodles</li>
    </ul>
  </body>
</html>
```

<div class="lesson-note">
  <h4>Designer Note</h4>
  <ul>
    <li>The skeleton is invisible to users — but it's critical infrastructure</li>
    <li>Missing doctype or charset breaks silently — correct in one browser, garbled in another</li>
    <li>Getting this right is the professional baseline, not optional</li>
  </ul>
</div>
