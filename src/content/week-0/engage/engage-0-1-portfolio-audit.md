## Overview

You'll pair up with a classmate and audit each other's DESN 368 portfolios. This isn't about criticism—it's about helping each other catch issues before they become problems.

By the end of this lab, you'll know exactly what needs fixing in your portfolio.

---

## Why This Matters

Your portfolio from DESN 368 might have:
- **Broken images** — Hosted on temp sites that expired
- **Dead links** — Netlify policy changes, moved repos
- **404 errors** — Projects that aren't deploying
- **Outdated info** — Old contact details, missing projects

You could find these issues yourself, but a fresh pair of eyes catches things you've learned to overlook.

## The Audit

Review your partner's portfolio using the checklist below. Be thorough but kind—you're helping them, not grading them.

### Checklist

**Access & Loading**
- [ ] Portfolio loads without errors (no 404, no blank page)
- [ ] Page loads in reasonable time (under 5 seconds)
- [ ] No browser console errors (open DevTools → Console)

**Images**
- [ ] All images display correctly
- [ ] No broken image icons (the little torn photo symbol)
- [ ] Images are appropriately sized (not blurry, not massive)

**Links**
- [ ] All navigation links work
- [ ] All project links work
- [ ] External links open correctly
- [ ] No links to localhost or broken Netlify URLs

**Projects**
- [ ] Each project page is accessible
- [ ] Project descriptions are present
- [ ] Project images/mockups display

**Responsiveness**
- [ ] Site works on mobile (check on your phone)
- [ ] No horizontal scrolling on mobile
- [ ] Text is readable without zooming

**Content**
- [ ] Contact info is current (if displayed)
- [ ] No placeholder text ("Lorem ipsum", "Coming soon")
- [ ] Name is spelled correctly
- [ ] Bio/about section exists (if applicable)
