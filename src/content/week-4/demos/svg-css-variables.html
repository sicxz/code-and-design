Approach 1: CSS Variables

SVG uses custom properties, changes with theme

<!-- SVG --> <svg class="icon-basic"> <path d="..." /> </svg> <!-- CSS --> .icon-basic { --icon-color: var(--color-text); } .icon-basic path { fill: var(--icon-color); transition: fill 300ms ease; }

Approach 2: currentColor

SVG inherits parent text color automatically

<!-- SVG --> <svg class="icon-current-color"> <path d="..." /> </svg> <!-- CSS --> .icon-current-color { color: var(--color-text); } .icon-current-color path { fill: currentColor; <!-- Inherits --> }

Approach 3: Multi-Part Icon

Different colors for different parts

<!-- SVG --> <svg class="icon-multipart"> <circle class="icon-background" ... /> <path class="icon-foreground" ... /> </svg> <!-- CSS --> .icon-background { fill: var(--color-surface); } .icon-foreground { fill: var(--color-accent); }

Approach 4: Outline + Fill

Stroke and fill independently themed

<!-- SVG (two layers) --> <svg class="icon-outlined"> <path class="fill-area" ... /> <path class="outline" ... /> </svg> <!-- CSS --> .outline { fill: none; stroke: var(--color-text); } .fill-area { fill: var(--color-accent); opacity: 0.2; }

Bonus: Hover Interaction

Hover over this card to change icon color

<!-- CSS --> .interactive-card:hover .icon-basic { --icon-color: var(--color-accent); } <!-- Icon color changes on card hover -->

System Integration

All icons respond to theme system

<!-- Design Tokens (root) --> :root { --color-text: #1a1a1a; --color-accent: #3498db; } [data-theme="dark"] { --color-text: #f5f5f5; --color-accent: #5dade2; } <!-- All icons inherit these tokens -->

🎯 Key Takeaways

  • CSS Variables: Most flexible, lets you change colors dynamically
  • currentColor: Simple, inherits text color (like icon fonts)
  • Multi-part: When different parts need different colors
  • Transitions: Add smooth theme switching (300ms ease)
  • No inline fills: Remove fill="#000" from SVG, use CSS instead
  • Design tokens: Define colors once, use everywhere (from Project 1!)
