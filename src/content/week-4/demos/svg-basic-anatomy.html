SVG Basic Anatomy Demo

Understanding viewBox, paths, and SVG structure

1. Understanding viewBox

The viewBox defines the coordinate system, while width and height define the display size.

Same viewBox, Different Display Size

<svg width="100" height="100" viewBox="0 0 100 100"> <circle cx="50" cy="50" r="40" /> </svg>

Same Coordinates, Scaled 2×

<svg width="200" height="200" viewBox="0 0 100 100"> <circle cx="50" cy="50" r="40" /> </svg>

Key insight: The circle's coordinates stay the same (cx="50" cy="50" r="40"), but it displays larger because the canvas is bigger. Think of viewBox as the "camera frame" and width/height as the "print size."

2. Basic SVG Shapes

SVG provides primitive shapes: circle, rect, line, and more complex paths.

Circle

<circle cx="60" <!-- center x --> cy="60" <!-- center y --> r="50" <!-- radius --> />

Rectangle

<rect x="10" y="10" <!-- top-left --> width="100" <!-- width --> height="100" <!-- height --> rx="10" <!-- corner radius --> />

3. The Path Element

The <path> element is the most powerful — it can draw anything using commands.

Triangle (Path)

<path d="M60 10 L110 100 L10 100 Z" /> Path Commands: M60 10 = Move to (60, 10) L110 100 = Line to (110, 100) L10 100 = Line to (10, 100) Z = Close path (back to start)

Star (Complex Path)

<path d="M60 10 L70 45 L105 45 L77 65 L87 100 L60 80 L33 100 L43 65 L15 45 L50 45 Z" fill="none" stroke="#9b59b6" stroke-width="2" />

Path commands:

  • M x y — Move to (doesn't draw)
  • L x y — Line to
  • H x — Horizontal line to x
  • V y — Vertical line to y
  • Z — Close path (line back to start)
  • C x1 y1, x2 y2, x y — Cubic Bézier curve (Figma uses this for curves)

4. Groups and Structure

The <g> element groups shapes together — like layers in Figma.

Bookmark Icon (Grouped)

<svg class="bookmark-icon" viewBox="0 0 24 24"> <g class="bookmark-group"> <path class="bookmark-body" d="M5 2 L19 2 L19 22 L12 18 L5 22 Z" /> </g> </svg> Why groups? - Apply transforms to multiple shapes - Organize complex icons - Target with CSS

5. viewBox as Camera Frame

Changing viewBox is like changing where the camera points.

Full View

<svg viewBox="0 0 100 100"> <!-- Shows 0,0 to 100,100 --> </svg>

Zoomed In (viewBox smaller)

<svg viewBox="25 25 50 50"> <!-- Shows 25,25 to 75,75 --> <!-- Same shapes, cropped view --> </svg>

Zoomed Out (viewBox larger)

<svg viewBox="0 0 200 200"> <!-- Shows 0,0 to 200,200 --> <!-- Same shapes, more space --> </svg>

Key takeaway: Shapes don't change coordinates, but viewBox changes what portion of the coordinate system you see.

💡 Key Concepts

  • viewBox = coordinate system (what the camera sees)
  • width/height = display size (how big it prints)
  • <path> = most flexible, can draw anything
  • <g> = groups shapes (like Figma layers)
  • Coordinates = always relative to viewBox, not pixels
