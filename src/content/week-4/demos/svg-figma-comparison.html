Figma Export Comparison

Same bookmark icon, three different approaches

Figma Raw Export Figma

File size: ~450 bytes
<!-- Generated by Figma --> <svg width="24" height="24" viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg" > <g id="bookmark-icon"> <g id="Layer_1"> <path id="Vector" d="M5.0000 2.0000 L19.0000... fill="#3498DB" /> </g> </g> </svg>

❌ Issues:

  • Generator comment (metadata)
  • xmlns attribute (not needed inline)
  • fill="none" on SVG (doing nothing)
  • Nested empty groups
  • Generic IDs (Vector, Layer_1)
  • Inline fill color (can't theme with CSS)
  • Excessive decimal precision (5.0000)

✅ Good parts:

  • Valid SVG code
  • Preserves Figma layer structure (can re-import)
  • Works correctly

Hand-Coded Manual

File size: ~120 bytes (73% smaller!)
<svg class="hand-coded" viewBox="0 0 24 24"> <path class="icon-shape" d="M5 2 L19 2 L19 22 L12 18 L5 22 Z" /> </svg> <!-- CSS (separate file) --> .icon-shape { fill: var(--icon-color); }

✅ Advantages:

  • No bloat — only what's needed
  • Semantic class name
  • No inline styles (CSS controls color)
  • Whole number coordinates (precise enough)
  • Easy to read and modify
  • Smallest file size

❌ Challenges:

  • Requires understanding path commands
  • Time-consuming for complex shapes
  • Not practical for detailed illustrations

SVGO + Manual Best

File size: ~180 bytes (60% smaller!)
<svg class="bookmark-icon" viewBox="0 0 24 24" role="img" aria-label="Bookmark" > <path class="bookmark-shape" d="M5 2h14v20l-7-4-7 4V2z" /> </svg> <!-- SVGO automatically: - Removed metadata - Removed xmlns - Removed fill="none" - Flattened groups - Simplified path (h,v shorthand) Manual additions: - Semantic class names - ARIA attributes -->

✅ Best of both:

  • Automated optimization (SVGO)
  • Semantic naming (manual)
  • Accessibility attributes (manual)
  • CSS-ready (no inline styles)
  • Small file size
  • Production-ready

🎯 Recommended approach:

  • Design in Figma (best design tool)
  • Export with optimal settings
  • Run through SVGO (automated cleanup)
  • Manual polish (classes, ARIA)

📊 File Size Comparison

  • Figma Raw: ~450 bytes (baseline)
  • Hand-Coded: ~120 bytes (73% smaller, but manual work)
  • SVGO + Manual: ~180 bytes (60% smaller, automated + polished)

🎯 The Workflow (Week 4 Approach)

  1. Design in Figma with code in mind:
    • Name layers semantically
    • Flatten groups
    • Outline strokes
    • Use simple shapes
  2. Export with optimal settings:
    • ☑ Outline stroke
    • ☑ Flatten
    • ☑ Simplify
  3. Run SVGO (automated cleanup)
  4. Manual polish:
    • Add semantic class names
    • Remove remaining inline styles
    • Add ARIA labels (if interactive)
    • Test in browser

💡 Key Takeaways

  • Figma is verbose by design — metadata helps re-import
  • SVGO automates 80% of cleanup — use it!
  • Manual polish adds the final 20% — semantic names, accessibility
  • Hand-coding is educational but not practical for complex shapes
  • The best approach: Figma prep + SVGO + manual cleanup
