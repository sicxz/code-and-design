SVG Interactive States Demo

Hover, focus, click, and keyboard-test all the examples below

1. Hover State

💡 Try: Hover over the button

<!-- HTML --> <button class="btn-hover"> <svg aria-hidden="true"> <path class="icon-shape" ... /> </svg> Add to Favorites </button> <!-- CSS --> .btn-hover { border: 2px solid #3498db; color: #3498db; } .btn-hover:hover { background: #3498db; color: white; } .btn-hover:hover .icon-shape { fill: white; }

2. Focus State (Keyboard)

⌨️ Try: Tab to button, see outline

<!-- CSS --> .btn-focus:focus-visible { outline: 3px solid #f39c12; outline-offset: 4px; transform: scale(1.05); } <!-- :focus-visible = keyboard only :focus = mouse + keyboard -->

3. Active/Click State

🖱️ Try: Click and hold the button

<!-- CSS --> .btn-active:active { transform: scale(0.95); background: #f5f5f5; } .btn-active:active .icon-shape { fill: #3498db; } <!-- :active = pressed down state -->

4. Toggle State (JavaScript)

🔄 Try: Click to toggle bookmark on/off

<!-- HTML --> <button class="btn-toggle" onclick="this.classList.toggle('active')" >... </button> <!-- CSS --> .btn-toggle .icon-shape { fill: none; stroke: #666; } .btn-toggle.active .icon-shape { fill: white; stroke: white; }

5. SVG Animation (Transform)

✨ Try: Hover to see icon rotate and scale

<!-- CSS --> .icon-link .icon-shape { transform-origin: center; transition: transform 300ms ease; } .icon-link:hover .icon-shape { transform: rotate(15deg) scale(1.1); } <!-- SVG transforms work on individual paths! -->

6. Loading State (Spinner)

⏳ Try: Click to simulate loading

<!-- CSS --> .btn-loading svg { display: none; } .btn-loading.loading svg { display: block; animation: spin 1s linear infinite; } @keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } <!-- JS toggles .loading class -->

♿ Accessibility Best Practices

  • Use :focus-visible — Shows outline for keyboard, not mouse
  • Add aria-label — For icon-only buttons (screen readers)
  • Use aria-hidden="true" — For decorative icons (with text)
  • Respect prefers-reduced-motion — Disable animations if requested
  • Ensure contrast — Focus outlines must be visible in all themes
  • Test with keyboard — Tab, Enter, Space should all work
  • Test with screen reader — VoiceOver (macOS) or NVDA (Windows)
