Practice 3-1: The Inheritance

Export your Figma variables to CSS

In-Class Lab — Part of Project 1: Signal & State

Overview

Time to bring your Figma variables into code. You'll create a variables.css file that mirrors your Figma collections, then build a working theme toggle.

This is where design meets development — your Figma decisions become real CSS.

Requirements

  • Create a variables.css file with your brand tokens
  • Refactor at least one page to use these tokens
  • Add a working theme toggle

From Figma to Code

Your Figma collections become your CSS:

Figma Collection CSS Section
color_primitives Color Tokens (primitives)
brand_colors Color Tokens (semantic)
typography Typography Tokens
spacing Spacing Tokens

Use your exact Figma variable names in your CSS. This creates a 1:1 relationship between your design file and your code.

Using the Export Plugin

  1. Install a plugin (each works to an extent, none are perfect):
    • Export Styles and Variables to CSS
    • CSS variables import/export
    • Variables to CSS
  2. Open your portfolio file in Figma (the one with your variables from Build 2-1)
  3. Make sure you're in Design mode (not Dev mode)
  4. Run the plugin: Right-click → Plugins → [Your chosen plugin]
  5. Export your tokens:
    • Copy the Variables output into a new variables.css file
    • Copy Styles into figma-styles-temp.css (optional reference)

Organizing Your Token File

Organize your exported variables into a clean structure:

/* Design Tokens — [Your Name]'s Portfolio */

/* Color Tokens */
:root {
  /* Your light mode colors go here */
}

/* Dark mode overrides */
[data-theme="dark"] {
  /* Same variable names, different values */
}

/* Typography Tokens */
:root {
  /* Your font families and sizes */
}

/* Spacing Tokens */
:root {
  /* Your spacing scale */
}

Key points:

  • :root holds your light mode (default) values
  • [data-theme="dark"] overrides only colors that change
  • Keep the same variable names your Figma export gave you

Step-by-Step Guide

Step 1: Audit Your Current CSS

Look at your existing stylesheets:

  1. How many different colors do you use?
  2. How many font sizes?
  3. Are values repeated in multiple places?

Step 2: Import Your Variables

Import your variables.css into your styles.css:

/* My Style Sheet */
@import "./variables.css";

This gives you:

  • Single source of truth
  • Predictable cascade
  • Reusable across pages and projects
  • Easy theming

Step 3: Refactor Your CSS

Find hardcoded values and replace them with variables:

Before:

body {
  background-color: #ffffff;
  color: #1a1a1a;
  padding: 32px;
}

After:

body {
  background-color: var(--color-surface-primary);
  color: var(--color-text-primary);
  padding: var(--space-xl);
}

Tip: This is a good opportunity to use GitHub Copilot. Ask it to "find hardcoded colors in my CSS and replace them with variables from variables.css"

Step 4: Upgrade the Theme Toggle

You already built a basic toggle in Practice 2-1: The Switch using classList.toggle. Now we upgrade it to use data-theme.

Add a toggle button:

<button class="theme-toggle" data-theme-toggle>
  Toggle Theme
</button>

Add the JavaScript (in main.js):

const toggle = document.querySelector('[data-theme-toggle]');

toggle.addEventListener('click', () => {
  const currentTheme = document.documentElement.dataset.theme;
  const newTheme = currentTheme === 'dark' ? 'light' : 'dark';
  document.documentElement.dataset.theme = newTheme;
});

Note: We're using data-theme on <html> instead of a class on <body>. This is the pattern we'll build on in Practice 3-2.

Step 5: Test Everything

  1. Click the toggle — do colors change?
  2. Check DevTools Elements panel — does data-theme appear on <html>?
  3. Check for console errors
  4. Verify your page looks correct in both modes

Complete Criteria

Your lab is complete when:

  • variables.css exists with at least 5 tokens
  • At least one page uses the tokens (no hardcoded colors)
  • Theme toggle works
  • No console errors

Naming Conventions

Semantic vs. Literal:

Literal (avoid) Semantic (prefer)
--color-blue --color-accent
--color-dark-gray --color-text-secondary
--font-size-16 --font-size-base

Semantic names describe purpose, not appearance. When your brand color changes from blue to green, --color-accent still makes sense.
