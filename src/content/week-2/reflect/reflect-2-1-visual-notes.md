# R&D // Weekly Visual Notes

## Week 2: User Preferences & Accessibility

------

### Objective

Create hand-written visual notes that capture key concepts, ideas, and learnings from this week's content.

------

### This Week's Focus

The web isn't one-size-fits-all. Users customize their systems for comfort, accessibility, and preference. This week's readings explore how CSS can detect and respect those choices — and why that matters.

------

### Required Reading

1. [prefers-reduced-motion: Sometimes less movement is more](https://web.dev/articles/prefers-reduced-motion) — web.dev (~10 min)
2. [prefers-color-scheme: Hello darkness, my old friend](https://web.dev/articles/prefers-color-scheme) — web.dev (~10 min)
3. [A Complete Guide to CSS Media Queries](https://css-tricks.com/a-complete-guide-to-css-media-queries/) — CSS-Tricks (focus on the **Preference Features** section) (~15 min for that section)

------

### Requirements

| Spec   | Details                           |
| ------ | --------------------------------- |
| Pages  | 2–4 pages (5.5" × 8.5" each)      |
| Medium | Moleskine Cahier notebook         |
| Tools  | Pen (any color that scans well)   |
| Style  | Hand-written with visual elements |

------

### What to Include

Document anything that helps you process and remember the material:

- Key concepts and definitions
- Quick sketches or diagrams
- Questions that came up
- Code snippets or structures
- Connections between ideas
- Personal insights or "aha" moments
- Resources you want to remember

**This week, pay special attention to:**

- What user preferences can CSS detect?
- Why does reduced motion matter? Who benefits?
- How do you balance respecting preferences with design intent?
- What assumptions have you been making about your users?

------

### Submission

1. Scan your pages using a document scanner or scanning app
2. Create a single PDF file (right-reading order)
3. Name your file: `week-2-notes-lastname.pdf`
4. Upload to Canvas by **Sunday at midnight**
