CSS APIScroll-Driven Animation Properties

animation-timeline scroll() | view() | --name Links a CSS animation to a scroll timeline instead of a clock. This is the core property. animation-timeline: view()
animation-range keyword% keyword% Controls when a view() animation starts and ends. Format: "[entry-keyword] [%] [exit-keyword] [%]". Keywords: entry, exit, cover, contain. animation-range: entry 0% cover 40%
animation-duration time (any) Still required by the spec when using animation-timeline, but its value is meaningless — scroll position controls timing. Set to 1ms and ignore it.
animation-fill-mode both | forwards | backwards Use both to keep the animated state at 0% and 100% scroll. Without this, the animation resets when scrolled back to start.
scroll-timeline-name --custom-name Applied to a scroll container. Names the scroll timeline so child elements can reference it. Must start with --. scroll-timeline-name: --my-scroller
scroll-timeline-axis block | inline | x | y Specifies the scroll direction for a named timeline. Use block for vertical scroll (most common), inline for horizontal.
view-timeline-name --custom-name Names a view timeline on an element so other elements can react to its visibility. view-timeline-name: --card-view
scroll() function Creates an anonymous scroll progress timeline. scroll(root) watches the page, scroll() watches the nearest scrollable ancestor.
view() function Creates an anonymous view progress timeline for the element — based on its visibility in the viewport. Each element gets its own independent timeline.

GSAPScrollTrigger Properties

trigger selector | element The element ScrollTrigger watches. When it crosses the start/end thresholds, actions fire. Defaults to the animated element if not set.
start "edge position" When the trigger zone activates. Format: "[trigger-edge] [viewport-position]". E.g. "top 80%" = element's top hits 80% down the screen.
end "edge position" | "+=px" When the trigger zone deactivates. Same format as start, or relative: "+=500" = 500px after the start point.
scrub boolean | number true = direct 1:1 link to scroll position. A number (e.g. 1) = animation catches up over that many seconds. Smoother feel.
pin boolean | selector true = pin the trigger element. A selector = pin a specific element. Switches to position: fixed for the trigger duration.
pinSpacing boolean | "padding" Whether GSAP adds padding to the parent to compensate for the pinned element's removal from flow. Default: true. Set false for side-by-side layouts.
toggleActions "e l re rl" Four values: onEnter onLeave onEnterBack onLeaveBack. Options: play | pause | resume | reverse | restart | reset | complete | none. "play none none reverse" is the most common.
toggleClass object Add/remove a CSS class when the trigger activates. { targets: ".el", className: "is-active" } — performant for state-based visual changes.
markers boolean Shows start/end/scrub marker lines for debugging. Set markers: true during development. Remove before shipping.
animation tween | timeline Attach a pre-built GSAP tween or timeline to this ScrollTrigger instance. Useful when creating the ScrollTrigger with ScrollTrigger.create() separately.
onEnter / onLeave function Callback functions that fire when the trigger zone is entered or exited. Use for side-effects like updating state, switching classes, or triggering external events.

GSAPstart / end Syntax

Common start values
start: "top 80%" // top of element hits 80% down screen start: "top center" // top of element hits screen center start: "top top" // top of element hits top of screen start: "bottom 80%" // bottom of element hits 80% down start: "center center" // midpoints align
Common end values
end: "bottom top" // element's bottom hits top of screen end: "bottom bottom" // element's bottom hits bottom of screen end: "+=500" // 500px after the start point end: "+=100%" // 100% of trigger element's height end: "+=2000" // 2000px — common for pin durations
Timeline position parameter (3rd arg)
tl.to(el, { ... }) // after previous ends (default) tl.to(el, { ... }, "<") // same time as previous starts tl.to(el, { ... }, "<0.2") // 0.2s after previous starts tl.to(el, { ... }, "-=0.3") // 0.3s before previous ends tl.to(el, { ... }, "+=0.5") // 0.5s after previous ends tl.to(el, { ... }, 0) // at absolute time 0
toggleActions reference
"play none none reverse" // most common "play none none none" // one-shot "play pause resume reverse" // full control "restart none none none" // restarts every entry // Available actions: // play | pause | resume | reverse // restart | reset | complete | none

PatternSVG Line Drawing

The setup (JS)
// 1. Select path and measure it const path = document.querySelector("#my-path"); const len = path.getTotalLength(); // 2. Hide the line by offsetting the dash pattern path.style.strokeDasharray = len; path.style.strokeDashoffset = len; // 3. Animate offset to 0 — line "draws" itself gsap.to(path, { strokeDashoffset: 0, scrollTrigger: { trigger: "#container", start: "top top", end: "bottom bottom", scrub: true } });
How the stroke trick works
// stroke-dasharray: the dash + gap pattern // Setting it to the full path length = one long dash // stroke-dashoffset: shifts the start of the pattern // Setting it equal to the length = dash pushed off screen // Setting it to 0 = full line visible // So: offset goes from [length] → 0 // Result: line appears to draw itself // Critical: always use getTotalLength() // Never hardcode the path length // Paths change; hardcoded values break silently

VocabGlossary

Motion Parallax
Perceptual depth cue where closer objects appear to move faster. Web parallax exploits this to imply spatial depth on flat screens.
Scrollytelling
Narrative technique where the story unfolds as the user scrolls. Scroll action drives content progression, not just page position.
Scroll-Driven Animation
Animation whose progress is mapped to scroll position rather than time. Scroll down = animate forward. Scroll up = animate backward.
Scrubbing
Linking animation progress directly to the scrollbar. The scrollbar becomes a playhead that controls a timeline forward and backward.
Pinning
Locking an element in the viewport for a set scroll distance. Content scrolls past or within it while it stays fixed on screen.
Scroll Progress Timeline
CSS timeline based on how far a scroll container has been scrolled. 0% = top, 100% = bottom. Used with scroll().
View Progress Timeline
CSS timeline based on a specific element's visibility in the viewport. Starts when element enters, ends when it exits. Used with view().
animation-timeline
Core CSS property that links an animation to a scroll source instead of a clock. Accepts scroll(), view(), or a named timeline.
prefers-reduced-motion
CSS media query that detects if the user has enabled "Reduce Motion" in system settings. Always implement this for scroll animations.
toggleActions
GSAP ScrollTrigger property defining behavior at four scroll events: onEnter, onLeave, onEnterBack, onLeaveBack.
Jank
Visible stuttering or frame drops during animation. Usually caused by animating layout-triggering properties (width, height, top, left) or main-thread blocking.
Scroll Hijacking
Overriding the browser's native scroll behavior. Breaks user expectations and accessibility. Never do it. Different from scroll linking.
