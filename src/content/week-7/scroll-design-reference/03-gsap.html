03

GSAP ScrollTrigger

The industry
standard.

When CSS isn't enough — complex sequences, pinned sections, cross-browser reliability — GSAP ScrollTrigger is the tool. Free since 2024. Runs everywhere. Think in timelines, not keyframes.

The case for GSAP

Why reach for
a library?

CSS handles many scroll patterns well. But it can't pin elements, it struggles with complex multi-element sequencing, and browser support for scroll-driven animations is still incomplete (especially Safari).

GSAP ScrollTrigger was built specifically for the patterns that CSS can't handle: pinning, scrubbing through a choreographed timeline, reversing on scroll-up, and synchronizing multiple animations with precise control.

Since Webflow acquired GSAP in 2024, it's completely free — including commercial use.

Setup — two lines

Load GSAP, load ScrollTrigger, register it once. That's the entire setup.

<!-- Load both from CDN --> <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script> <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js"></script> // Register the plugin — do this once, before any animations gsap.registerPlugin(ScrollTrigger);

Concept 01

Cross a line.
Fire the animation.

The simplest use: an element enters the viewport, the animation runs. Scroll up enough and the cards below will have reset — scroll back down to watch them animate in again.

Feature Lead

The Trigger

Define a DOM element. When it crosses the threshold you set, your animation fires. Nothing else required.

Mechanism

Threshold Logic

start and end define the active zone in human-readable format: "top 80%" means the element's top edge hits 80% down the screen.

Control

toggleActions

Four values control what happens at each scroll boundary: enter, leave, re-enter, re-leave. "play none none reverse" is the most common.

// Shorthand: element triggers itself when it enters view gsap.from(".trig-card", { scrollTrigger: { trigger: ".trig-cards", start: "top 80%", // cards' top edge hits 80% down the screen toggleActions: "play none none reverse" }, opacity: 0, y: 35, duration: 0.6, stagger: 0.12, // each card delays slightly ease: "power2.out" });

Reading start and end

Two words.
Two positions.

The start and end values follow a consistent pattern. Read each one as two words: the trigger element's edge, then the viewport position where it crosses.

← viewport top (start: "top top")
← trigger element top (enters here)
← viewport center (start: "top center")
← trigger element bottom
viewport bottom
"top 80%" → element's top edge hits 80% down the screen
most common for reveal animations
"top top" → element's top edge reaches the top of the screen
used for pin: true sequences
"bottom center" → element's bottom reaches the center
good for staggered list items
"+=500" → 500px after the start point
relative end for pinned sections

toggleActions

Four events.
Four responses.

toggleActions takes four values, each corresponding to a different scroll event. Read them left to right as the four moments a scroll trigger can fire.

toggleActions: "onEnter onLeave onEnterBack onLeaveBack" // ↓ ↓ ↓ ↓ // Valid values: play | pause | resume | reverse | restart | reset | complete | none
Most common
"play none none reverse"
Plays forward on enter. Reverses when you scroll back past the top edge. The "feels natural" default.
One-shot
"play none none none"
Fires once and stays. The animation doesn't reverse when you scroll back up.
Full control
"play reverse play reverse"
Plays when entering from either direction, reverses when leaving from either direction.
Scrub replaces this
scrub: true
When scrub is set, toggleActions is ignored. Scrub directly links animation progress to scroll position — no events needed.

How scrub works

Scroll position drives
animation progress.

gsap.timeline({ scrollTrigger: { trigger: "#scrub-demo", start: "top 60%", end: "bottom 40%", scrub: 1 // 1 second lag for smoothness } }) .to("#sw1", { x: 0, duration: 1 }) .to("#sw2", { x: 0, duration: 1 }, "<0.2") .to("#sw3", { x: 0, duration: 1 }, "<0.2");
scrub: trueDirect 1-to-1 link. The animation progress equals scroll progress instantly. Can feel mechanical.
scrub: 1The animation "catches up" over 1 second. Feels smoother, more cinematic. Higher numbers = more lag.
"<0.2"Timeline position offset. "<" means "start at the same time as the previous tween, offset by 0.2 seconds." Creates overlapping reveals.
When to use scrubWhen scroll controls the story — product walkthroughs, data reveals, parallax sequences. Not for simple entrance animations (use toggleActions instead).

Step 01 / The pin

The left card stays.
You scroll the right.

The card is pinned using CSS position: sticky. It remains visible as you scroll through all three steps. This is how Apple product pages hold a device image in view while feature text scrolls by.

Sub-copy and the call-to-action complete the card. Three scroll steps. One visual. Multiple triggers. This is the complete scrollytelling pattern — deconstructed into its parts.

The pin pattern — deconstructed

Two triggers.
One experience.

The pin pattern separates concerns: one trigger holds the visual in place; separate triggers fire changes inside it as the reader scrolls through each chapter.

// TRIGGER 1: Pin the visual for the full section duration ScrollTrigger.create({ trigger: "#pin-stage", start: "top top", end: "bottom bottom", pin: "#pin-left", // what gets pinned pinSpacing: false // don't add padding (layout-dependent) }); // TRIGGER 2: Fire changes on each step gsap.timeline({ scrollTrigger: { trigger: "[data-step='1']", start: "top center", toggleActions: "play none none reverse" } }) .to("#pin-title", { opacity: 1, y: 0, duration: 0.5 }) .to("#pin-sub", { opacity: 1, y: 0, duration: 0.4 }, "<0.15");
pin: "#pin-left"The element to pin. It gets position: fixed while the trigger zone is active, then releases when you scroll past.
pinSpacing: falseGSAP adds padding by default to compensate for the pinned element. Set false when you're using a side-by-side layout with its own height.
Separate triggersThe pin trigger and the chapter triggers are independent. The pin handles position; the chapter triggers handle content state. Clean separation of concerns.
toggleClass alternativeInstead of animating properties directly, you can use toggleClass to add/remove CSS classes on the visual — a clean, performant way to manage visual state changes.
Common bug: the layout jump

When pin activates, GSAP switches the element to position: fixed, which removes it from normal flow. GSAP adds padding to the parent to compensate. If your layout jumps when pinning, check: (1) the parent has a defined height, (2) pinSpacing is set correctly, (3) no conflicting CSS transform on the pinned element.

GSAP Timelines

Sequence. Overlap.
Choreograph.

A GSAP timeline is a container for multiple animations. You add tweens to it — each one relative to the previous — and control the whole sequence with a single ScrollTrigger.

let tl = gsap.timeline({ scrollTrigger: { trigger: ".scene", pin: true, scrub: 1, start: "top top", end: "+=2000" // 2000px of scroll controls the whole timeline } }); // Add animations — each runs AFTER the previous by default tl.to(".box-a", { x: 400, rotation: 360 }) .to(".box-b", { y: 200, scale: 1.5 }) .to(".box-a", { backgroundColor: "#c93820" }); // runs after box-b finishes // Position parameter: when to start relative to the timeline tl.to(".box-c", { x: 100 }, "+=0.5") // 0.5s after previous ends .to(".box-d", { x: 100 }, "<") // same time as previous .to(".box-e", { x: 100 }, "<0.2"); // overlap by 0.2s
tl.to().to().to()Animations chain sequentially by default. Each runs after the previous ends. The whole chain is mapped to the scroll distance defined in scrollTrigger.
Duration as proportionWith scrub, tween duration values define the proportion of scroll distance each tween gets. Longer duration = takes more scroll to complete.
Position parameterThe third argument to .to() controls timing relative to the timeline. "<" = same start. "<0.2" = offset by 0.2. "+=0.5" = 0.5 after end. "-=0.3" = 0.3 before end.
