Why bother?

Scroll is the most natural input on the web.

Every user knows how to scroll. Scroll-driven design doesn't add a new interaction — it gives meaning to one that already exists.

Two paths.

Native CSS or JavaScript library?

Modern CSS handles many scroll effects natively and with better performance. GSAP handles everything else — complex sequences, cross-browser reliability, pinning.

The risk.

Motion that harms is never a design choice.

Parallax and scroll animations can trigger vestibular disorders. Responsible implementation requires a reduced-motion fallback — full stop.

A brief history

From Flash to
native CSS.

2000s

The Flash Era

Timeline-based animations inside a browser plugin. Immersive, but inaccessible, plugin-dependent, and dead on mobile.

Plugin
2012

NYT "Snow Fall"

The New York Times published a landmark avalanche story combining scroll-triggered animations, video, and text. It named scrollytelling as a genre and proved it could serve journalism.

Milestone
2017

IntersectionObserver API

Browsers added a way to detect when elements enter the viewport without expensive scroll event listeners. Better for triggering animations, but not for syncing them to scroll position.

API
2018+

GSAP ScrollTrigger

GreenSock released ScrollTrigger, which became the industry standard for complex scroll-controlled animation — pin, scrub, sequence, full timeline control.

Library
2023–

Native CSS Scroll-Driven Animations

The W3C spec stabilized. Browsers shipped animation-timeline. Simple scroll effects now need zero JavaScript — and run off the main thread.

Current

Three terms you need first.

Core vocabulary
Parallax
Layers moving at different speeds during scroll. The perceptual illusion of depth on a flat screen — borrowed from how we see the physical world.
Scrollytelling
A narrative that unfolds as the user scrolls. The scroll action drives story progression: text reveals, visuals evolve, sequences advance.
Scroll-Driven Animation
The mechanism behind both. Animation progress is mapped directly to scroll position — not a timer. Scroll down: forward. Scroll up: back.
"Scroll-driven animations have evolved from janky, main-thread JavaScript implementations to smooth, accessible, off-main-thread experiences using modern CSS and UI features like Scroll Timelines and View Timelines." — Chrome for Developers, NRK Case Study
