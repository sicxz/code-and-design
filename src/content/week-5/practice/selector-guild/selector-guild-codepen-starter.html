Chapter 1: The City of Markup

In Markup, every building has a tag name. The grand tower is an h1. The chapter signs are h2 elements. The ordinary citizens are paragraphs, moving through the page with quiet structure.

A selector is a search spell. It tells CSS exactly who should change. Some spells find every element of one kind. Others find only the elements living inside a particular neighborhood, like paragraphs inside .bio.

Chapter 3: The Link Portal

The selector apprentices discover that links have states. One link is new, one has been visited, one reacts to the mouse, one responds to the keyboard, and one changes while being clicked.

Chapter 4: The Selector Timeline

  • Type selector: finds every matching tag, like h1, h2, or section.
  • Class selector: finds every element with a shared class name, like .timeline.
  • Descendant selector: finds an element only when it lives inside another element, like .bio p.
  • Pseudo-class selector: finds an element in a specific state, like a:hover or a:focus.

Chapter 6: The Box Model Castle

Every element lives in a box. The content sits in the center. Padding creates space inside the walls. The border marks the edge. Margin creates space between one element and the next.

When you style section, header, and main, you are not just changing colors. You are revealing the invisible structure of the page.
