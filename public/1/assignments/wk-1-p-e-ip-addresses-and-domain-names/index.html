Part 1 — Find the IP Address of a Domain

Tool

https://www.site24x7.com/tools/find-ip-address-of-web-site.html

Steps

  1. Enter a website you actually use.
  2. Click Find IP Address.
  3. Add a note card with the IP address(es) returned.
  4. Add a screenshot of the result.

Questions to Answer

(Bullet points are fine)

  • What IP address(es) came back?
  • Why might a domain have multiple IPs (CDNs, load balancers, regions)?
  • How does/might an IP relate to the physical server(s)?

Part 2 — Geolocate an IP

Tool

https://www.iplocation.net/ip-lookup

Steps

  1. Enter 171.67.215.200 (a Stanford University IP).
  2. Click Lookup IP Address.
  3. Add a screenshot showing the results of multiple providers.

Questions

  • What location is associated with this IP?
  • What factors might cause discrepancies in IP geolocation data?

Part 3 — Find Your Public IP

Steps

  1. Open Google
  2. Search what is my IP and screenshot the result.
  3. (Optional) Verify/details at WhatIsMyIP.com.

Questions

  • What is your public IP?
  • IPv4 or IPv6?
  • Can you approximate your location using the Part 2 tool?
  • Why does it matter that your IP is publicly visible (privacy, tracking, rate-limits, security)?

Reflection (150–300 words)

Create a note card in the Reflection section.

Address

  • How Domain Names Map to IPs and the Role of DNS.
  • What IPs can reveal (location, ISP/organization) and what they can't (your exact address).
  • Privacy/security implications (VPNs, shared IPs, school/work NATs, logging).

Bonus (Optional)

In 2–4 sentences, explain DNS caching (browser → OS → resolver → authoritative) and how Time to Live (TTL) affects when changes appear.
