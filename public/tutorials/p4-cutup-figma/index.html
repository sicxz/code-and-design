The Deal

Why this tutorial exists

You have two jobs in P4: design it in Figma, then build it in code. Most of the class is stuck between those two jobs — the Figma isn't quite done, and the code feels like a mountain.

This tutorial shows you the shortcut: when your Figma file is actually finished — layers named, sections grouped, motion notes written — you can point an AI coding agent (opencode) at it through the Figma MCP, and the agent will read your real layer structure and write a P4-compliant semantic HTML5 scaffold in about 15 minutes.

This is different from pasting a Figma URL into a chat. With MCP, the agent queries the Figma API directly — it sees your actual frame names, layer hierarchy, and component instances. That means cleaner code, no hallucinated sections, and far less back-and-forth.

The catch AI can't read what isn't there. If your Figma layers are named Frame 47, Rectangle 12, Group 3 — the cut-up will be slop, because that's what the MCP sees. The cleaner your Figma, the cleaner the code. Finishing the design IS doing the code.

Today in class you'll watch one finished Figma get cut up live. Once your own Figma is ready, run the four steps below yourself and submit the result as the P4 Cut-Up Checkpoint before Tuesday.

Setup — one-time, ~10 min

opencode Desktop + Figma MCP

You'll set up three things, and none of it needs the terminal: download the opencode Desktop app, get a Figma personal access token, and paste that token into opencode so it can read your Figma. Everything happens in your browser, in Finder, and in the opencode app — no code editor, no config files, no commands.

1. Download and install opencode Desktop

  1. Go to opencode.ai/download.
  2. Download the Mac app. Apple Silicon Macs (M1-M4, basically anything recent) get opencode-desktop-mac-arm64.dmg; older Intel Macs get opencode-desktop-mac-x64.dmg.
    Not sure which you have? Apple menu → About This Mac. If the chip says "Apple M-something," you're on Apple Silicon.
  3. Double-click the downloaded .dmg, then drag OpenCode Desktop into your Applications folder.
  4. Open it from Applications (or Launchpad). That's it — no commands to type.
"opencode can't be opened because Apple cannot check it for malicious software"? That's just macOS Gatekeeper being cautious about a fresh download. Right-click (or Control-click) the app in Applications → Open → Open. You only have to do this once.

2. Get a Figma personal access token

  1. Go to figma.com/settings → Security tab.
  2. Scroll to Personal access tokens → "Generate new token."
  3. Name it desn368-p4-cutup. Scope: leave the defaults (File content read access is required).
  4. Copy the token immediately (starts with figd_). You won't see it again.
Treat this token like a password. Don't paste it into Discord, don't share it. You'll paste it straight into opencode in the next step — it stays on your machine.

3. Make your project folder

Still no terminal — just Finder. Right-click on your Desktop → New Folder → name it p4-landing-page. That's the whole step. opencode will create your index.html inside this folder for you in a moment — you don't make any files by hand.

4. Open the folder in opencode and paste your Figma key

  1. In OpenCode Desktop, choose Open Folder / Open Project and pick your p4-landing-page folder. On first launch it asks which model provider to use — pick whichever you have access to (Anthropic, OpenAI, OpenRouter, etc.).
  2. Open opencode's Settings → MCP / Connections and add the Figma connection.
  3. When it asks for a key, paste your figd_ token — the one you copied in Step 2 — and save. No config files, no editing — just paste it in.

opencode connects to Figma with that key. You should see figma listed as connected in the MCP / Connections panel before you go on.

Already pay for ChatGPT or Claude? Use those instead. The exact same workflow works with Codex (ChatGPT Plus / Pro) and Claude (Claude Pro / Max) — and both ship a desktop app you just download, exactly like opencode, so you still never open the terminal:
  • Codex: download the Codex desktop app, sign in with your ChatGPT account, then add the Figma MCP from the app's Settings → Connectors / MCP.
  • Claude: download the Claude desktop app, sign in with your Claude account, then add the Figma MCP from Settings → Connectors.
All three read Figma the same way once the Figma MCP is connected. The prompts below work unchanged — just paste your Figma token / access when the app asks for it.
Figma readiness check Before you start the cut-up, your Figma should have:
  • Layer names that mean something (Hero, Feature Card, Nav Link) — not Frame 47.
  • Frames grouped by section (hero / video / features / data / contact).
  • Motion annotations written as text labels next to the elements (e.g., "button lifts on hover").
The MCP shows the agent your real layer names. Garbage names in = garbage HTML out.
01 Prime opencode with the brief + your Figma file

One setup message — loads the rules and the design

Grab your Figma file's URL from the browser (the part that looks like figma.com/design/<FILE_KEY>/<file-name>). You'll paste it into the priming message below. Opencode will use the MCP to call the Figma API and read your real layer structure — no guessing.

Open the opencode chat and paste this as your very first message:

You are helping me cut a Figma design into a P4 landing page for DESN 368. Hold the following P4 rules in context for every reply that follows. == P4 REQUIRED STRUCTURE == Outer: <header id="header"> <img id="header-img" alt="..."> <nav id="nav-bar"> <a class="nav-link">...</a> (minimum 3 nav-link anchors) </nav> </header> <main> ... </main> <footer> ... </footer> Inside <main>, exactly five sections in this order: <section id="hero"> -- value proposition + primary CTA <section id="video"> -- embedded video <section id="features"> -- repeating <article> cards <section id="data"> -- contains <table id="data-table"> <section id="contact"> -- contact form == TAG RULES == - Use <article> for repeating self-contained items (feature cards). - Use <section> for thematic groupings with a heading. - Every <img> needs descriptive alt text (5-10 words). Never alt="". - Every form <input> has a paired <label for=id>. No placeholder-as-label. - Email input: type="email" required. - Submit button: id="submit". - Data table: id="data-table" with <thead> and <tbody>. - The data table must scroll horizontally on mobile (wrap in a div with overflow-x: auto). == MOTION == Wrap @keyframes animations in @media (prefers-reduced-motion: no-preference). == MY DESIGN == Here is my Figma file: [PASTE YOUR FIGMA URL] Use the Figma MCP (get_figma_data or equivalent) to read the file. Then, based on the real layer/frame names you find, do the mapping. == CONFIRMATION == Do not write any HTML yet. First, reply with: 1. The 5 section IDs you'll put inside <main>, in order, mapped to the actual top-level frames you found in my Figma. 2. The required IDs/classes you'll bake into the scaffold (list them). 3. Which Figma frames/components you intend to map to <article> tags (cite their real layer names from the file). 4. Any Figma frame that does NOT fit the P4 required structure (so I know what to cut or rework on the design side). 5. Any motion annotations or text labels you saw in the Figma that describe animation or hover behavior.
Watch opencode call the MCP. You'll see opencode invoke the Figma MCP tool right in the chat — it'll show the tool name and its arguments. Approve the call. The agent then has your real layer structure, not a guess.
Read the agent's reply line by line. Verify it cites real layer names from your file (not generic placeholders), lists all five section IDs in the right order, every required ID/class above, and that the article mapping matches what's actually in your Figma. If anything is missing or wrong, say so before moving on — for example: "You missed id='header-img'. Add it." or "I don't actually have a video section. Drop it from the plan and tell me what I'm missing."
Why this priming step matters: Without it, every later prompt has to re-state the P4 rules and re-paste the Figma URL — tedious and easy to forget one. With it, you can write prompts like "scaffold the HTML" or "build the form" in one short line, and the agent already knows the rules and has direct access to your design. You only do this priming once per opencode session.
02 The Constraint Cut

Generate the scaffold

Now that opencode has the P4 rules in context and a live read on your Figma via the MCP, asking for the scaffold is one short prompt. Here's the checklist you'll verify the output against — you don't need to paste it into the prompt:

WhereRequired selectorNotes
Outer header<header id="header">Fixed position
Logo imageid="header-img"Inside the header
Nav element<nav id="nav-bar">Inside the header
Each nav linkclass="nav-link"Minimum 3 links
Each feature card<article>…</article>Inside #features
Data table<table id="data-table">With <thead> + <tbody>
Form submit buttonid="submit"Inside the contact form

Now ask opencode to scaffold:

Using the rules and the map you confirmed, write the full HTML for index.html in this folder. Start with DOCTYPE, lang="en", viewport meta, <title>, and a link to css/styles.css. Pull real text from my Figma via the MCP for headlines, button labels, and nav link labels — don't invent placeholder text if the real content is in the file. Write descriptive alt text on every image based on what each image actually shows in Figma. HTML only. No CSS yet. Write the file directly using your file-edit tool.
The agent writes the file directly. Opencode has file-editing tools. You'll see it propose changes to index.html — approve them and the file appears on disk. No copy-paste needed. Open index.html in your editor or browser and verify the seven items in the table above are present. Cmd/Ctrl + F for each ID.
What the AI gets wrong here:
  • Wraps everything in <div>s instead of <section> / <article>.
  • Forgets alt attributes, or writes lazy alt like alt="image".
  • Uses class="header" instead of id="header".
  • Skips the <footer> entirely.
Fix these now. Don't move to Step 3 until the scaffold is clean. One follow-up prompt — "You used divs instead of section tags inside main. Rewrite using the required semantic structure." — usually does it.
03 The Logic Implementation

Build the table and the form

The data table and the contact form are the two pieces graders look at hardest, because they have the strictest structural requirements. Do them in their own prompt so the agent doesn't shortcut.

The Data Table

Inside <section id="data">, replace any placeholder with a real <table> using the columns and rows shown in my Figma data section. Read those columns and rows from the Figma file via the MCP if they're laid out as text in the design. Use <th scope="col"> for the header row. Wrap the table in <div class="table-scroll" style="overflow-x: auto;"> so it scrolls on mobile. (Apply the rest of the P4 table rules you already have.) Update index.html directly. Don't change anything else in the file.
Why horizontal scroll? Your table has 4-6 columns of product specs. On a phone, stacking each row into a card is a real design choice — but it requires custom CSS per column. Horizontal scroll is the responsible default: it preserves the tabular reading and it works the moment you ship. The modal version stays optional. If you want it, do it after the checkpoint.

The Contact Form

Inside <section id="contact">, build the contact form using the P4 form rules you already have (paired label/input, type="email" required, id="submit", no placeholder-as-label). Match the fields shown in my Figma — check the form section via the MCP for the actual field labels. Use action="#" for now (no backend). Update index.html directly.
What the AI gets wrong here:
  • Uses placeholder="Email" as a label substitute. Placeholders are not labels — screen readers don't read them as labels and they disappear when the user types.
  • Forgets the for/id pairing between <label> and <input>.
  • Misses type="email" — you lose the free browser validation.
04 The Motion Audit

Translate Figma motion notes into CSS

If you wrote motion annotations on your Figma during the design phase ("button lifts on hover," "hero text fades in"), opencode saw them when it called the MCP. The brief asks for one @keyframes animation and one transform property. Pick the two most important.

Look at the motion notes/annotations from my Figma (you saw them via the MCP). Implement two of them: 1. ONE @keyframes animation matching one note, applied to the relevant element by class. 2. ONE transform on :hover (or :focus) matching another note, applied to the relevant element by class. Create css/styles.css if it doesn't exist and link it from index.html if the link isn't there already. Use class names that already exist in my HTML — don't invent new ones. Respect prefers-reduced-motion (per the rule you already have). Write the files directly.
No motion notes in your Figma? The agent will tell you. Add one or two simple ones to your Figma (a sticky-note next to the button saying "hover: lift 4px" is enough) and re-run the prompt. Or, just describe the two animations in the prompt itself: "animate the hero heading with a 600ms fade-in on load, and lift the primary CTA 2px on hover."
Read your output

What the AI gets wrong (the short list)

Opencode is fast, but it's also confident when it's wrong. The whole point of being a designer-coder is that you read the output. Here's the short list of failures that show up most often. Spot them, fix them, and submit only after.

FailureHow to spot itFix
Divs instead of semantic tags Cmd-F for <section — should hit 5+ times Re-prompt with the exact required tags
Missing or lazy alt text Cmd-F for alt="" or alt="image" Rewrite alt to describe what's in the image, in 5-10 words
Placeholder used as label No <label for="..."> in form HTML Add a real label per input, pair with for/id
Wrong ID on submit button Cmd-F for id="submit" One re-prompt: "Submit must have id='submit'"
Class used where ID was required Cmd-F for id="header", id="nav-bar" Re-prompt with required ID list
Section order doesn't match your Figma Compare scaffold to the top-to-bottom flow of your design "Reorder the sections in main to match the Figma top-to-bottom."
Skipped the MCP entirely Agent invented frame names or content that aren't in your file "Call get_figma_data on my file first, then redo the mapping using the real layer names."
The submission asks you to name ONE thing the AI got wrong. Pick the one that mattered most for your project and write 2-3 sentences in your prompt log about what it was, how you noticed, and what you changed. Generic answers ("it forgot some tags") don't count; specific ones ("it skipped id='header-img' on the logo image, I caught it by Cmd-F'ing for header-img, and I re-prompted with the exact requirement") do.
